# Word 解决方案文档规范

生成中文解决方案类 Word 文档时，严格遵循以下规范。适用于 OpenClaw 和 Claude Code。

## 何时触发

- 用户要求生成/编写 Word 文档、解决方案文档
- 用户提到"写个方案"、"出个文档"、"生成 word"

## 排版规范

### 正文

- 字体：仿宋，小四（12pt）
- 首行缩进 2 字符
- 段落行距：1.5 倍

### 标题层级

| 级别 | 序号格式 | 字体 | 字号 | 行距 | 缩进 |
|------|---------|------|------|------|------|
| 一级 | 一、二、三、四… | 黑体 | 二号（22pt） | 1.5 倍 | 无缩进 |
| 二级 | 1、2、3… | 黑体 | 小二（18pt） | 1.5 倍 | 无缩进 |
| 三级 | 1.1、1.2、1.3… | 黑体 | 三号（16pt） | 1.5 倍 | 无缩进 |
| 四级 | 1.1.1、1.1.2、1.1.3… | 黑体 | 小三（15pt） | 1.5 倍 | 无缩进 |
| 五级 | 1.1.1.1、1.1.1.2… | 黑体 | 四号（14pt） | 1.5 倍 | 无缩进 |

**关键规则：**
- 所有序号采用**多级列表**形式，序号自动递增
- 一级标题的序号**不在后续标题中继承和显示**（即二级标题用"1、2、3"而非"一.1、一.2"）
- 各级标题均无首行缩进

### 表格

- 根据窗口自动调整大小
- 所有文字居中对齐
- 标题行（首行）文字加粗
- 表格文字统一：仿宋，五号（10.5pt）
- 表格边框：所有单元格 0.5pt 黑色实线（顶/底/左/右/内部横/内部竖共 6 方向）

## 文档结构

1. **首页**（单独一页）：文档标题、项目信息等
2. **目录**（单独一页）：自动生成目录
3. **正文**：按标题层级组织内容

### 页码

- 页码从目录页开始计算
- 首页无页码

## 生成方式

使用 Python `python-docx` 库生成。核心脚本见 `scripts/generate.py`。

### 快速使用

```bash
python scripts/generate.py --output 输出路径.docx --title "文档标题"
```

或通过 Markdown 输入：

```bash
python scripts/generate.py --input 内容.md --output 输出路径.docx
```

### 在对话中生成

直接告诉 AI："按 word-solution-doc 规范生成 Word 文档"，AI 会调用本 skill 的脚本生成。
